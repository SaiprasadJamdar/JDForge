# JDForge: Voice-to-JD Intelligence Engine

An advanced, end-to-end recruiter automation engine that effortlessly converts your raw multilingual hiring conversations, technical syncs, or video meetings into perfectly separated Job Descriptions and highly optimized Boolean Search Queries using Groq's blindingly fast LLM infrastructure.

## Core Features
- **Intelligent Media Intake**: Automatically processes both pure audio formats (`.wav`, `.mp3`) and full video meetings (`.mp4`, `.mkv`), seamlessly leveraging `FFmpeg` to extract audio under the hood.
- **Verbatim Multilingual Preservation**: Captures local unstructured hybrid dialects perfectly using native non-biased Groq Whisper (`whisper-large-v3-turbo`).
- **Smart English Normalization**: Passes raw speech through a highly structured cleanup prompt using `llama-3.3-70b-versatile` to fix broken grammar and normalize distinct regional terms into a clean, ATS-friendly English baseline.
- **Multi-Role Orchestration**: Effortlessly scans a single 30-minute sync transcript and detects if multiple distinct job profiles were discussed, dynamically splitting the context correctly.
- **Dynamic Artifact Generation**: For *each* detected role, the system simultaneously outputs a highly polished Job Description AND a targeted, recruiter-ready Boolean Search Query.
- **Collision-free Taxonomy**: Extracts strict role titles and hashes collision-free IDs (e.g., `data_analyst_1a2b3c.txt`) keeping all synchronous files instantly searchable across workflows.

## Setup Instructions

1. Ensure FFmpeg is installed and accessible in your system's PATH (required for video intake).
2. Create a `.env` file in the root directory:
   ```env
   GROQ_API_KEY=your_key_here
   ```
3. Install dependencies:
   ```bash
   pip install -r requirements.txt
   ```
4. Boot the Interactive Engine:
   ```bash
   python main.py
   ```

## Architecture & Lifecycles

The data lifecycle automatically populates and enforces the following directory structures:
- `recordings/` – Holds native microphone recordings generated by the tool or manually dropped meeting videos/audios. 
- `raw_transcripts/` – Output of verbatim, untampered multilingual Whisper extraction.
- `clean_transcripts/` – Output of the normalized ATS-friendly English translation standard.
- `jd_outputs/` – The generated polished Job Description text files.
- `search_queries/` – The generated technical Boolean Search strings perfectly correlated by filename to the JD.

## Interacting with the CLI
- **Option 1 (Record)**: Press `X` at any time to immediately safely halt recording and save to the library.
- **Option 2 & 3 (Process)**: Interactively handles queue management. Enter single numbers (`1`), lists (`1, 3`), or just press `Enter` to batch-process everything dynamically!
